# いちれん不動産 サイト ― 独自ドメイン（ichirenfudosan.com）設定ガイド

サイト内のURLはすべて **https://ichirenfudosan.com/** に変更済みです。
あとは「GitHub側の設定」と「DNS（ドメイン）側の設定」の2つを行えば、独自ドメインで表示されます。

---

## 全体の流れ（3ステップ）
1. GitHubにこのフォルダの中身をアップ（`CNAME` ファイルも忘れず一緒に）
2. ドメインのDNSにレコードを追加
3. GitHub の Settings → Pages でドメインを設定 → HTTPS を有効化

---

## ステップ1：GitHubにアップ
- このフォルダの中身でリポジトリを置き換えてプッシュ。
- **重要**：`CNAME` という名前のファイル（中身は `ichirenfudosan.com` の1行）を
  必ずリポジトリの一番上の階層に置いてください。これが独自ドメインの目印になります。

## ステップ2：DNSにレコードを追加
ドメイン `ichirenfudosan.com` のDNS設定画面を開きます。

### まず「DNSをどこで設定するか」を確認する（Google Workspace の場合）
Google Workspace（有料Gmail）で使っているドメインは、契約時の登録会社のDNS画面で設定します。確認手順：

1. **Google管理コンソール** <https://admin.google.com> に管理者アカウントでログイン。
2. 左メニュー **［アカウント］→［ドメイン］→［ドメインの管理］** を開く。
3. 対象ドメイン `ichirenfudosan.com` の登録事業者（レジストラ）名が表示されます。
   - 「Google Domains」「Squarespace」と出る → **Squarespace のドメイン管理画面**でDNSを設定（※Google DomainsはSquarespaceに移管済み）。<https://account.squarespace.com> にログイン → Domains → 該当ドメイン → DNS Settings。
   - 「お名前.com」「Xserver」など他社名が出る → その会社の管理画面のDNS設定で行います。
4. ※ もし「Google Domains」と表示されてログイン先が分からない場合は、Gmailで「Squarespace ドメイン移管」の案内メールを検索すると、引き継ぎ先アカウントが分かります。

> ＝ Workspace（Gmail）の管理画面はメール用で、Web表示のDNSは上記「登録会社」側で設定する、と覚えてください。

### 設定先が分かったら、以下のレコードを追加
※ ホスト名/名前の入力欄に「@」が使えない画面では、空欄のままか、ドメイン名そのものを入れてください（画面により表記が異なります）。

#### A) ルートドメイン（ichirenfudosan.com）を使う ＝ 推奨
以下の **Aレコード4つ** を追加します（ホスト名／名前は「@」または空欄）：

| 種類 | 名前(ホスト) | 値 |
|------|------|------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

可能なら IPv6 の **AAAAレコード4つ** も追加（任意・推奨）：

| 種類 | 名前 | 値 |
|------|------|------|
| AAAA | @ | 2606:50c0:8000::153 |
| AAAA | @ | 2606:50c0:8001::153 |
| AAAA | @ | 2606:50c0:8002::153 |
| AAAA | @ | 2606:50c0:8003::153 |

さらに「www付き」でもアクセスできるように、**CNAMEレコード**を1つ：

| 種類 | 名前 | 値 |
|------|------|------|
| CNAME | www | take1ren-star.github.io |

### ⚠ メール（Gmail / Google Workspace）は触らないこと
**MXレコード**（メール用。`ASPMX.L.GOOGLE.COM` などGoogle宛のもの）は絶対に削除・変更しないでください。
今回追加するのは Web 表示用の A / AAAA / CNAME だけです。MX を残しておけば Gmail はそのまま使えます。

## ステップ3：GitHub側でドメインを設定
1. リポジトリ → **Settings** → 左メニュー **Pages**。
2. 「Custom domain」に `ichirenfudosan.com` と入力して Save。
3. DNSが反映されると（数十分〜最大1日）、下に緑のチェックが出ます。
4. **「Enforce HTTPS」にチェック**を入れる（鍵マーク＝暗号化。出てこない場合は反映待ち、少し待って再確認）。

---

## 反映の確認
- ブラウザで <https://ichirenfudosan.com/> を開く → サイトが出ればOK。
- 反映には DNS の浸透待ちで **数十分〜最大24時間** かかることがあります。
- 鍵マーク（https）が付くまでも少し時間がかかる場合があります。

## 仕上げ：Google Search Console
ドメインが変わったので、検索登録もやり直します。
1. <https://search.google.com/search-console> で `https://ichirenfudosan.com/` を登録。
2. 「サイトマップ」→ `sitemap.xml` を送信。
3. 「URL検査」にトップURLを入れて「インデックス登録をリクエスト」。

### 困ったとき

- 「www あり」をメインにしたい、リダイレクトしたい等のご希望があれば調整します。
